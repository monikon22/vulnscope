# VulnScope

[English](README.md) | Українська

VulnScope — це сканер вразливостей з пріоритетом TUI для автоматизованого виявлення поширених вразливостей вебдодатків.

## Особливості

- Робочий процес з пріоритетом TUI з панеллю керування, потоком нового сканування, прогресом живого сканування, історією сканування та налаштуваннями.
- Безпечне DAST-сканування з обмеженнями області дії, обмеженням швидкості, таймаутами та неруйнівними корисними навантаженнями.
- Визначені користувачем профілі сканування зі збереженими лімітами запитів та вибором окремих правил.
- Рушій YAML-правил з реєстрами правил (`web`, `fingerprints`) та імпортом користувацьких правил.
- Віддалені стрічки правил з локальним сервером стрічок (`vulnscope serve`) та локальним кешем з урахуванням хешів.
- Локальна історія сканування в SQLite.
- Детермінована оцінка ризиків з модифікаторами впевненості та експозиції.
- Звіти у форматах HTML, JSON та Markdown.
- Виявлення компонентів за допомогою заголовків, мета-тегів та шаблонів JS-активів.

## Порівняння

| Особливість | VulnScope | Традиційний DAST (напр., ZAP, Burp) | Сканери на основі шаблонів (напр., Nuclei) |
| :--- | :--- | :--- | :--- |
| **Основний інтерфейс** | **TUI-first** (Інтерактивний термінал) | Важкий GUI / Web UI | Тільки CLI |
| **Модель безпеки** | **Safe-by-Design** (Тільки GET, жодних руйнівних дій) | Повний спектр (може бути руйнівним при неправильному налаштуванні) | Залежить від шаблонів (може бути агресивним) |
| **Визначення правил** | **Декларативний YAML** (На основі реєстрів) | Скриптовий / Ручний / На основі плагінів | YAML-шаблони |
| **Розподіл правил** | **Віддалені стрічки** (Синхронізація на основі хешів) | Ручні оновлення / Магазин плагінів | Спільнотні YAML-репозиторії |
| **Стратегія обходу** | BFS зі суворим виконанням області дії | Складний спайдерінг / Ручне проксіювання | Зазвичай на основі списку цілей (без глибокого обходу) |
| **Створення відбитків** | Інтегроване виявлення компонентів на основі YAML | Евристичний / Ручний | Виявлення на основі шаблонів |
| **Інтеграція CI/CD** | Безголовий CLI $\rightarrow$ JSON/HTML/MD | Керування через API / Важкі контейнери | Легкий CLI $\rightarrow$ JSON/Text |
| **Керування станом** | Локальний репозиторій SQLite (Відтворюваний) | На основі сесій / Файли проекту | Зазвичай без стану (вивід у файл) |

## Швидкий старт

```bash
pipx install vulnscope
vulnscope
```

З вихідного коду:

```bash
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
vulnscope
```

## Базове використання

```bash
vulnscope
vulnscope scan https://example.local
vulnscope doctor
vulnscope update
vulnscope serve --ip 127.0.0.1 --port 8080
vulnscope serve --path ./rules
vulnscope import ./report.json
```

## Правила

Локальні правила знаходяться в `rules/` і групуються за реєстром (наприклад, `rules/web` та `rules/fingerprints`).

Віддалені стрічки надають індекс та сирі сторінки YAML-правил. Клієнт кешує віддалені індекси та правила в `~/.vulnscope/cache/remote-feeds/` (або кастомному шляху кешу через перевизначення CLI).

Дивіться [документацію з правил](docs/ua/rules.md).
