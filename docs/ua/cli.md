# CLI

`vulnscope` відкриває панель керування TUI.

Команди:
- `vulnscope scan <url>`: відкрити TUI з попередньо заповненою ціллю.
- `vulnscope import <file>`: імпортувати історію сканування JSON.
- `vulnscope doctor`: перевірити середовище виконання, SQLite, правила та каталог звітів.
- `vulnscope update [--cache-dir <path>]`: перевірити налаштовані віддалені стрічки та повідомити кількість виявлених локальних/віддалених правил.
- `vulnscope serve [--ip] [--port] [--path <dir>]`: запустити локальний вебсервер, який публікує реєстри правил та сирі YAML-правила.
- `vulnscope profiles ...`: команди для перегляду/створення/видалення/встановлення профілю за замовчуванням.

Поведінка `vulnscope serve`:
- `GET /` з `Accept: application/json` повертає JSON-індекс реєстрів та правил.
- `GET /` без JSON accept повертає стилізований HTML-індекс із пошуком.
- Посилання на правила відкривають сирий YAML, коли клієнт запитує YAML/JSON, в іншому випадку — стилізовану HTML-сторінку з кнопкою копіювання.
- JSON стрічки включає хеші правил та повний хеш стрічки для перевірки кешу на стороні клієнта.

Кеш віддалених стрічок:
- Корінь кешу за замовчуванням: `~/.vulnscope/cache/remote-feeds`.
- `vulnscope update --cache-dir ./cache` перевизначає корінь кешу для конкретного запуску команди.
