# Документація VulnScope

VulnScope — це сканер вразливостей з пріоритетом TUI, орієнтований на безпечні перевірки вебсайтів та YAML-правила на основі реєстрів.

## З чого почати

1. `docs/architecture.md` — повна архітектура та потік даних.
2. `docs/scanner.md` — як працює конвеєр безпечного сканування.
3. `docs/rules.md` — схема правил та механізми зіставлення.
4. `docs/security-model.md` — межі безпеки та обмеження.

## Карта документації

- `docs/architecture.md`: компоненти, межі та діаграми послідовності.
- `docs/scanner.md`: область дії, обхід (crawl), перевірки корисного навантаження, події сканування.
- `docs/rules.md`: YAML-схема, реєстри, віддалені стрічки (feeds), зіставлення.
- `docs/reports.md`: експорт у HTML/JSON/Markdown та структура даних.
- `docs/tui.md`: екрани, навігація, робочі процеси профілів, гарячі клавіші.
- `docs/configuration.md`: повний YAML + перевизначення через змінні оточення.
- `docs/cli.md`: команди CLI та опції.
- `docs/development.md`: робочий процес розробки та тестування.
- `docs/deployment.md`: локальне розгортання та розгортання в контейнерах.
- `docs/security-model.md`: обмеження безпеки за замовчуванням.

## Типові сценарії

### Перший запуск

1. Налаштуйте `vulnscope.yaml` (або використовуйте значення за замовчуванням).
2. Запустіть `vulnscope doctor`.
3. Відкрийте TUI (`vulnscope`) та запустіть Нове сканування (New Scan).
4. Перегляньте результати та експортуйте звіт.

### Створення правил

1. Додайте YAML-правило в `rules/<registry>/`.
2. Перевірте завантаження за допомогою `vulnscope update`.
3. За потреби запустіть локальну стрічку за допомогою `vulnscope serve`.
4. Запустіть сканування та перевірте звіти.

### Командна робота

1. Стандартизуйте набори профілів у `profiles`.
2. Підключіть централізовану віддалену стрічку.
3. Використовуйте експорт у JSON для автоматизації та HTML/MD для звітності.
